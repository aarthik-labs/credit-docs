## 1) Install


```npm
pnpm add @aarthiklabs/credit-sdk@0.0.4-1
```

```npm
yarn add @aarthiklabs/credit-sdk@0.0.4-1
```

## 2) Get your keys

From Aarthik you’ll receive:

* tenant_id
* application_id
* API key (server-only)
* Base URL (hosted): https://credit.aarthiklabs.com

You must provide:

* Your parent origin (example: https://app.yourcompany.com)

Ensure this origin is allowlisted; it will be sent as `allowed_parent_origin` when minting bootstrap tokens.

<Callout type="info" title="Self-hosted or local API?">
See [Tenant Onboarding Guide](./docs/tenant-onboarding) for required env vars and local bootstrap setup.
</Callout>

<Callout type="warning" title="Never expose the API_KEY in the browser">
If the API_KEY appears in your frontend bundle, your integration is compromised.
Always mint Bootstrap Tokens from your backend.
</Callout>

## 3) Mint a Bootstrap Token from your backend

Your backend should expose a minimal endpoint like:

* `POST /api/credit/bootstrap-token`

This endpoint:

1. Validates the borrower identity (your auth)
2. Calls Aarthik’s Bootstrap Token API using the API key (server-to-server)
3. Returns `{ bootstrap_token }` to the frontend

### Example: Next.js App Router Route Handler

Create:

`app/api/credit/bootstrap-token/route.ts`

```ts
import { NextResponse } from "next/server";

export const runtime = "nodejs";
export const dynamic = "force-dynamic";

export async function POST() {
  try {
    // 1) Server-to-server: call Aarthik Credit API using API key
    const baseUrl =
      process.env.AARTHIK_API_BASE_URL ?? "https://credit.aarthiklabs.com";
    const apiKey = process.env.AARTHIK_API_KEY;
    const tenantId = process.env.AARTHIK_TENANT_ID;
    const applicationId = process.env.AARTHIK_APPLICATION_ID;
    const allowedParentOrigin = process.env.AARTHIK_ALLOWED_PARENT_ORIGIN;

    if (!apiKey || !tenantId || !applicationId || !allowedParentOrigin) {
      return NextResponse.json(
        { error: "Missing server configuration" },
        { status: 500 }
      );
    }

    // Keep allowed_parent_origin server-defined. Do not accept it from clients.
    const res = await fetch(
      `${baseUrl}/api/tenants/${tenantId}/applications/${applicationId}/auth/bootstrap`,
      {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
          "x-api-key": apiKey,
        },
        body: JSON.stringify({
          allowed_parent_origin: allowedParentOrigin,
        }),
      }
    );

    if (!res.ok) {
      const text = await res.text().catch(() => "");
      return NextResponse.json(
        { error: "Failed to mint bootstrap token", details: text },
        { status: 502 }
      );
    }

    const data = await res.json();

    // Expect shape: { bootstrap_token: string, ... }
    if (!data?.bootstrap_token || typeof data.bootstrap_token !== "string") {
      return NextResponse.json(
        { error: "Invalid token response shape" },
        { status: 502 }
      );
    }

    return NextResponse.json(
      { bootstrap_token: data.bootstrap_token },
      { status: 200, headers: { "Cache-Control": "no-store" } }
    );
  } catch (err) {
    return NextResponse.json(
      { error: err instanceof Error ? err.message : "Unknown error" },
      { status: 500 }
    );
  }
}
```

## 4) Frontend: open the lending journey

You’ll typically:

1. User clicks “Apply for a loan”
2. Your frontend calls your backend (`/api/credit/bootstrap-token`)
3. You pass the `bootstrap_token` into the SDK component and open it

A simple pattern is to fetch the token on click and store it in state.
